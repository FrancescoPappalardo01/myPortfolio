Portfolio Webpage Project

Project Description

During my bootcamp, I developed a personal portfolio website to showcase the projects I completed throughout the program. The primary goal of this project was to create a visually appealing, user-friendly platform where visitors can view my work and easily contact me.

I used HTML and CSS to build and style the main content of the webpage, ensuring it was responsive and accessible across various devices. The design focuses on clean, modern aesthetics with a layout that highlights my completed projects, each accompanied by descriptions and links to view the source code.

For the contact section, I implemented JavaScript to perform client-side form validation, enhancing the user experience by ensuring correct data is submitted before the form can be processed. This feature ensured that visitors could easily reach out with inquiries or feedback.

Key Features

  • Project Showcase: A dedicated section highlighting individual projects completed during the bootcamp, each including descriptions, technologies used, and links to GitHub repositories.
  • Responsive Design: The portfolio is designed to be compatible across different screen sizes using media queries, ensuring accessibility on both mobile and desktop devices.
  • Interactive Contact Form: JavaScript is used for real-time form validation, providing instant feedback to users about form input errors.

Main Challenge

The biggest challenge was building an interactive and user-friendly website from scratch. It required careful planning to structure the site’s navigation intuitively and design an engaging user interface. By iterating through several design prototypes, I was able to create a polished and functional final product.

Calculator

Project Description

The objective of this project was to test and demonstrate my JavaScript knowledge by developing a functional calculator. This involved writing JavaScript functions, setting up event listeners, and creating a user interface with HTML and CSS.

Technologies Used

  • HTML: For structuring the calculator layout.
  • CSS: For styling the calculator to make it visually appealing and user-friendly.
  • JavaScript: For implementing the calculator’s functionality, including arithmetic operations and user interactions.

Key Features

  • Four Basic Operations: Supports addition, subtraction, multiplication, and division.
  • AC (All Clear) Button: Resets the calculator to its initial state, clearing all input and results.
  • Delete Button: Removes the last entered digit or character, allowing users to correct mistakes.
  • Percentage Button: Calculates the percentage of the current number.

Main Challenge

The main challenge was ensuring accurate and responsive functionality for all calculator operations. This involved handling various edge cases, such as division by zero and multiple consecutive operations, and managing the display updates correctly.

Weather API Web App

Project Description

The objective of this project was to create a simple web-based weather application to demonstrate my understanding of asynchronous JavaScript. The project retrieves real-time weather data from an external API and displays it to the user. It served as a learning opportunity to work with Promises and try-catch blocks for error handling.

Technologies Used

  • HTML: For structuring the layout of the weather app.
  • CSS: For styling the app and ensuring a user-friendly interface.
  • JavaScript: For handling the API calls, managing the data asynchronously, and implementing error handling with try-catch.

Key Features

  • City-based Weather Search: Users can input a city name to retrieve current weather information, including temperature, humidity, and conditions.
  • Real-time Weather Data: The app fetches live data from a public weather API and updates the display accordingly.
  • Error Handling: Includes error handling to manage failed API requests and invalid user inputs using try-catch.

Main Challenge

The main challenge was efficiently handling asynchronous requests to the API and managing various errors such as invalid inputs and network failures. Understanding how to work with Promises and implementing proper error handling with try-catch helped overcome these obstacles.
